# /intent-assess

> 评估 IDD 是否适合你的项目，学习 Intent 驱动开发

## 用法

```bash
/intent-assess
/intent-assess --learn    # 专注于 IDD 学习
```

## 目的

在采用 IDD 之前，此命令帮助你：

1. **评估项目适配性** - IDD 是否适合这个项目？
2. **学习 IDD 方法论** - 理解其理念和实践
3. **确定采用路径** - 如何在现有项目中引入 IDD

## 何时使用

- 开始新项目并考虑使用 IDD
- 为现有代码库评估 IDD
- 为团队成员介绍 IDD 概念
- 在 IDD 和传统方法之间做决定

## 项目适配标准

### 适合 IDD 的项目

| 因素 | 为什么有帮助 |
|------|-------------|
| **新功能/模块** | 从零开始进行 Intent 优先设计 |
| **复杂业务逻辑** | Intent 在编码前澄清需求 |
| **团队协作** | Intent 作为共享契约 |
| **长期维护** | Intent 在开发者离职后仍然存在 |
| **AI 辅助开发** | LLM 与结构化 Intent 配合更好 |

### 不太适合 IDD 的项目

| 因素 | 原因 |
|------|------|
| 快速原型 | 开销不值得 |
| 一次性脚本 | 没有长期价值 |
| 纯重构 | 不需要新的 intent |
| 琐碎改动 | 足够简单，不需要正式流程 |

## 输出

命令提供：

1. **适配评估**
   - 基于项目特征的评分
   - 具体建议

2. **学习材料**（使用 `--learn`）
   - IDD 理念概述
   - 与 TDD/SDD 的比较
   - 最佳实践

3. **下一步**
   - 推荐运行的命令
   - 针对你情况的采用策略

```
从一个新功能开始使用 IDD，然后扩展。

2. 为你的下一个功能运行 `/intent-interview`
```
