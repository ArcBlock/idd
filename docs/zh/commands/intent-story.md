# /intent-story

> 分享你的 IDD 采用故事

## 用法

```bash
/intent-story
/intent-story --lang zh    # 中文输出
/intent-story --lang en    # 英文输出
```

## 目的

通过结构化访谈，创建关于 Intent 驱动开发经验、经验教训和最佳实践的博客文章或文章。

## 何时使用

- 使用 IDD 完成了一个项目
- 想与社区分享学习心得
- 撰写内部案例研究
- 为 IDD 知识库做贡献

## 访谈流程

命令进行结构化访谈，涵盖：

### 1. 背景

- 你用 IDD 做了什么项目？
- 团队规模和经验如何？
- 之前使用什么开发方法？

### 2. 采用之旅

- 是什么促使你尝试 IDD？
- 你如何向团队介绍它？
- 最初的反应是什么？

### 3. 体验

- 什么效果好？
- 什么有挑战？
- 有什么意外？

### 4. 结果

- 有可衡量的改进吗？
- 团队反馈如何？
- 你会再次使用吗？

### 5. 建议

- 给其他采用 IDD 的人什么建议？
- 你会做什么不同的事？

## 输出格式

### 博客文章

```markdown
# 我们如何为 [项目] 采用 Intent 驱动开发

*作者 [作者] | [日期]*

## 挑战

[来自访谈的背景]

## 为什么我们尝试 IDD

[来自访谈的动机]

## 我们的旅程

[来自访谈的采用故事]

## 我们学到了什么

[来自访谈的经验教训]

## 结果

[来自访谈的成果]

## 给其他人的建议

[来自访谈的建议]

---
*此故事使用 IDD /intent-story 创建*
```

### 案例研究

更正式的格式，包含指标和结构化章节。

### 快速帖子

更短的格式，适合社交媒体或内部更新。

## 示例会话

```
> /intent-story

让我们记录你的 IDD 故事！

## 背景

你用 IDD 做了什么项目？
> 我们的支付处理系统重构

团队有多大？
> 4 名开发者

在 IDD 之前你们用什么方法？
> 传统的用户故事配合 Jira

## 采用

是什么让你尝试 IDD？
> 我们一直在产品和工程之间有误解。
> "真相来源" 的 Intent 概念吸引了我们。

你是如何引入的？
> 从一个新模块作为试点开始

## 体验

什么效果好？
> 结构化访谈流程很棒。它迫使我们在写代码之前
> 思考边界情况。
> TDD Agent 发现了我们会遗漏的问题。

什么有挑战？
> 让团队写 ASCII 图需要练习。
> 对"更多文档"有一些抵触

## 结果

有可衡量的改进吗？
> 代码审查中 bug 减少 40%
> 有了 Intent 文档，新开发者入职快多了

## 建议

你会告诉其他开始使用 IDD 的人什么？
> 从小处开始。选一个新功能。
> 不要试图一次性改造所有东西。

---

生成故事...

输出：blog/idd-story-payment-system.md

预览故事并根据需要编辑。
```

## 多语言支持

```bash
/intent-story --lang zh
```

生成中文故事，自然撰写（非翻译）。

## 输出文件

| 语言 | 文件 |
|------|------|
| 英文 | `blog/idd-story-[project]-en.md` |
| 中文 | `blog/idd-story-[project]-zh.md` |

## 发布

生成的故事可以：
- 发布到你的博客
- 内部分享
- 提交到 IDD 社区
- 添加到项目文档
